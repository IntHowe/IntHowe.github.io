Git基础操作

1 创建版本库

git init
在需要创建版本库的文件夹下执行“git init”命令即可，git会在当前目录下创建“.git”目录来跟踪管理版本库，并且创建一个"master"分支

把一个文件放到Git仓库只需要两步

git add []
将指定文件添加至版本库中的暂存区。

git commit [-m "提交说明"]
将暂存区所有内容提交至当前分支,"commit"命令可以一次提交很多文件，可以多次使用"add"命令多次添加文件，一次提交。

2 仓库管理

git status
查看仓库当前状态。

git diff
查看仓库具体修改内容。

git log [--graph] [--pretty = oneline] [--abbrev-commit]
查看提交日志，输出版本号(commit_id)等，参数"--graph"可以看到分支合并图，参数"--pretty = oneline"可以简化输出信息，参数"--abbrev-commit"可以缩略提交检验码。

git reset [--hard] <HEAD^>
将版本回退至上一个版本，git中用"HEAD"表示当前版本，上一版本用"HEAD"表示，上上个版本用"HEAD^"表示，也可用如"HEAD~100"回到上100个版本，也可用"git reset --hard commit_id"回退至指定版本，版本号可不写全。

git reflog
可查看每次执行的命令。

git checkout --
撤销file文件在工作区的全部修改，回到和暂存区一致的状态。

git reset HEAD
除了回退版本外，也可以把暂存区的修改回退到工作区，使用HEAD表示最新版本。

git rm
从版本库中删除该文件。

3 远程仓库

配置GitHub仓库

  1. 创建SSH Key
    在用户主目录下，查看是否有".ssh"目录，若有，查看是否有"id_rsa"和"id_rsa.pub"这两个文件，若有，则已经配置过。否则，打开Git Bash，创建SSH Key：

$ ssh-keygen -t rsa -C "youremail@example.com"
一路回车，使用默认值即可。成功后会创建".ssh"目录和"id_rsa"（私钥）、"id_rsa.pub"（公钥）这两个文件。

  1. 添加公钥
    登录Git Hub，打开"Account settings","SSH Keys"页面，点"Add SSH Key"，填上Title，在Key文本框中粘贴"id_rsa.pub"文件内容，点"Add Key"。

##关联远程仓库
从零开发最好是先创建远程库，然后从远程库克隆。

  1. 登录GitHub，创建仓库
  2. 克隆一个本地库

git clone <仓库地址>

先有本地仓库，后有远程仓库的关联方法

  1. 登录GitHub，创建仓库
  2. 关联一个远程仓库

git remote add origin <仓库地址>
"origin"是远程仓库默认名字

  1. 推送更新

git push [-u] origin [分支名] [tagname] [--tags] [:refs/tags/]
将指定的分支推送到远程仓库对应的分支，第一次推送加上"-u"参数可以把本地指定的分支和远程仓库对应的分支关联起来，以后推送可以省略该参数。参数"[tagname]"表示推送指定标签至远程服务器。参数"[--tags]"表示将所有标签信息推送到远程仓库。参数"[:refs/tags/]"表示删除远程标签。

git remote [-v]
查看远程库信息。参数"-v"表示显示更详细的信息，如果没有推送权限，就看不到push的地址。

git remote rm
解除本地库和远程库的绑定关系，删除远程库仍需登录GitHub进行删除

4 分支管理

git checkout [-b] <分支名>
切换至已有分支。"-b"参数表示创建并切换到新分支，相当于两个命令"git branch <分支名>"和"git checkout <分支名>"

git branch [-d] [-D] [<分支名>] [--set-upstream-to=origin/<远程分支> <本地分支>]
查看所有分支，并在当前分支前面标一个"*"号。如果有参数"<分支名>",则表示创建该分支。
如果有参数"-d"和"<分支名>"，则表示删除该分支。如果有参数"-D"和"<分支名>"，则表示强制删除该分支，在该分支没有被合并时使用。参数"[--set-upstream-to=origin/<远程分支> <本地分支>]"表示将本地指定的分支与远程指定的分支进行关联。

git merge [--no-ff -m "提交说明"] <分支名>
将指定的分支合并到当前分支。如果有冲突，git会通过"<<<<<<<","=======",">>>>>>>"标记出冲突中不同分支的内容，通过命令"git status"可以查看冲突，并手动解决冲突后再提交。
git默认使用"Fast forward"模式，在该模式下删除分支后会丢掉分支信息，就看不出曾经做过合并。通过参数"[--no-ff -m "提交说明"]"会使用普通模式合并，创建新的commit，所以需要添加新的提交说明。

git switch [-c] <分支名>
切换至已有分支。"-c"参数表示创建并切换到新分支。

git stash [list] [apply [<指定的现场>]] [drop] [pop]
储藏工作现场，在需要切换分支但是在当前分支并不想提交时使用。参数"[list]"表示列出现场列表。
参数"[apply [<指定的现场>]] "表示恢复现场但是不删除stash内容。参数"[drop]"表示删除stash内容。参数"[pop]"表示恢复现场并删除stash内容。

git cherry-pick <commit_id>
复制一个指定的提交到当前分支。

git pull
抓取更新。

git rebase
将本地未push的分叉提交历史整理成直线，使查看历史提交更容易。

4 标签管理

git tag [-d] [] [commit_id] [-a <表情名> -m <标签说明> [commit_id]]
查看标签。参数"[]"则表示为打标签命令，默认打在最新提交的commit上。参数"[] [commit_id]"表示在指定的commit上打标签。参数"[-a <表情名> -m <标签说明> [commit_id]]"表示创建带有说明的标签。参数"[-d] []"表示删除标签。

git show
查看标签信息
