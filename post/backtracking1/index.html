初识回溯算法

2021-04-08

回溯算法实际上一个类似枚举的搜索尝试过程，主要是在搜索尝试过程中寻找问题的解，当发现已不满足求解条件时，就 “回溯” 返回，尝试别的路径。

回溯算法框架如下：

result = []
def backtrack(路径, 选择列表):
    if 已不满足求解条件:
        return  //发现已不满足求解条件就回溯
    if 满足结束条件:
        result.add(路径)  //满足条件加入结果集
        return
    for 选择 in 选择列表:  //枚举
        backtrack(路径, 选择列表) //搜索尝试

leetcode #401. 二进制手表

对手表的灯选出n个灯进行一个全排列，每选出一颗灯相当于小时数或分钟数增加对应的数，那么选择过程中如果已经不符合时间要求（小时>11或分钟>59就不需要继续选灯了）就直接回溯，满足则加入结果集。

for循环枚举点亮灯的情况，i从index枚举到10，每次枚举

  • 减少一个需要点亮的灯数量num - 1
  • 从当前已点亮的灯后面选取下一个要点亮的灯 i + 1
  • 在hour中增加当前点亮灯的小时数，如果i大于3，当前灯是分钟灯而不是小时灯，则加上0个小时
  • 在minute中增加当前点亮灯的分钟数，如果i没有大于3，当前灯是小时灯而不是分钟灯，则加上0分钟
class Solution {
    int[] h = new int[]{0,0,0,0,0,0,1,2,4,8};  //小时灯对应的数值
    int[] m = new int[]{1,2,4,8,16,32,0,0,0,0};  //分钟灯对应的数值
    List<String> res = new ArrayList<>();  //结果集
    public void backtrack(int num,int index,int hour,int minter) {
        if(hour > 11 || minter > 59) {
            return;  //不满足条件的就直接回溯返回
        }
        if(num == 0) {  //剩余需要点灯的数目为0则表示以选出n个灯
            StringBuilder sb = new StringBuilder();
            sb.append(hour);
            sb.append(":");
            if(minter < 10) {
                sb.append("0");
            }
            sb.append(minter);
            res.add(sb.toString());  //满足条件则加入结果集
            return;
        }
        for(int i = index;i < 10;i++) {
            /**
             * 枚举搜索尝试
             * 参数：剩余需要点亮灯的数量，从索引index往后亮灯，当前小时数，当前分钟数
             */
            backtrack(num - 1,i + 1,hour + h[i],minter + m[i]); 
        }
    }

    public List<String> readBinaryWatch(int num) {
        backtrack(num,0,0,0);
        return res;
    }
}
